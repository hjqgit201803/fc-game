# QQ 飞车风格游戏设计文档

**项目**: fc-game - Python 版 QQ 飞车风格竞速游戏
**日期**: 2026-04-18
**类型**: 游戏设计规范

---

## 概述

基于 Python 开发的 2D 俯视视角赛车游戏，模仿 QQ 飞车的核心玩法。单人 Demo 版本，专注于漂移机制和速度感。

### 核心特性
- 2D 俯视视角赛车游戏
- 纯竞速模式，计时挑战
- 经典漂移机制（漂移+喷射）
- 单赛道 Demo
- 基于 Pygame 开发

---

## 核心模块设计

### Car 类
**职责**: 封装赛车的所有行为和状态

**属性**:
- `x, y` - 车辆位置
- `angle` - 车辆朝向角度
- `speed` - 当前速度
- `drift_state` - 漂移状态 (NONE, DRIFTING, NITRO)
- `nitro_level` - 漂移蓄力值 (0-100)
- `velocity_x, velocity_y` - 速度向量

**方法**:
- `update(dt)` - 每帧更新位置和状态
- `accelerate()` - 加速
- `brake()` - 刹车/倒车
- `turn_left()` - 左转
- `turn_right()` - 右转
- `start_drift()` - 开始漂移
- `end_drift()` - 结束漂移并触发喷射

### Physics 类
**职责**: 处理物理计算和碰撞检测

**方法**:
- `apply_friction(car)` - 应用摩擦力和空气阻力
- `check_collision(car, track)` - 检测车辆与赛道边界碰撞
- `calculate_drift_physics(car)` - 漂移时的特殊物理计算
- `update_position(car, dt)` - 根据速度更新位置

### Track 类
**职责**: 定义赛道边界和检测

**属性**:
- `boundaries` - 赛道边界点列表
- `checkpoints` - 检查点（用于检测完成圈数）
- `start_pos` - 起点位置和方向

**方法**:
- `is_on_track(x, y)` - 检测位置是否在赛道内
- `draw(surface)` - 绘制赛道
- `get_start_position()` - 返回起点坐标和角度

### Game 类
**职责**: 游戏主循环和状态管理

**状态枚举**:
- `MENU` - 菜单/标题画面
- `PLAYING` - 游戏进行中
- `FINISHED` - 完成一圈，显示成绩

**方法**:
- `run()` - 主循环入口
- `handle_events()` - 处理输入事件
- `update(dt)` - 更新游戏状态
- `draw()` - 渲染画面

### UI 类
**职责**: 绘制 HUD 和界面元素

**方法**:
- `draw_hud(surface, car, time)` - 绘制抬头显示
- `draw_speed()` - 显示当前速度
- `draw_drift_bar()` - 显示漂移蓄力条
- `draw_timer()` - 显示当前用时
- `draw_menu()` - 绘制菜单
- `draw_results()` - 绘制成绩画面

## 漂移机制

### 状态机

```
                    按住 Shift + 转向
        ┌─────────────────────────────────┐
        │                                 │
        ▼                                 │
    [正常] ──────────────────────► [漂移中]
        ▲                                 │
        │                                 │
        │            漂移过久/碰撞         │
        │◄────────────────────────────────┘
        │
     松开 Shift
        │
        ▼
    [喷射] ────────► [正常]
        (短暂加速后)
```

### 漂移参数

| 参数 | 值 |
|------|-----|
| 漂移触发键 | Shift (转向时按住) |
| 漂移角度偏移 | 速度方向与车身角度分离 15-30° |
| 漂移摩擦 | 比正常摩擦小，保持速度 |
| 漂移蓄力速度 | 每帧 +2 点 |
| 最大蓄力值 | 100 |
| 喷射持续时间 | 0.5 秒 |
| 喷射速度加成 | +30% 基础速度 |

### 视觉反馈
- **漂移时**: 车身轻微左右抖动
- **轮胎痕迹**: 漂移时留下粒子轨迹
- **漂移条**: HUD 显示渐变色蓄力条
- **喷射**: 车尾显示火焰效果

## 赛道设计

### Demo 赛道布局

```
─────────────────────
│                   │
│  ╭───────╮        │
│  │       │╲       │  START
│  │   ━━━━│ ╲  ━━━━│  ▲
│  │       │  ╲     │  │
│  ╰───────╯   ╲    │
│               ╲   │
│                ╲  │
│                 ━━│  FINISH
│                   │
─────────────────────
```

### 实现
- 使用赛道边界点定义可行驶区域
- 碰撞检测：检测车辆中心点是否越界
- 起点和终点线定义
- 赛道宽度约 80-100 像素

### 简化方案（初版）
- 矩形赛道：外矩形 (1200x800) 减去内矩形 (800x400)
- 碰撞检测：矩形边界检查
- 圆角处理：四个角落用圆弧连接

## 游戏流程

```
启动程序
    │
    ▼
显示标题画面
    │
    │ (按 SPACE)
    ▼
[游戏进行中]
    │
    │ (完成一圈)
    ▼
显示成绩画面
    │
    ├─ (按 R) ─► 重新开始
    │
    └─ (按 Q) ─► 退出程序
```

## 控制方式

| 按键 | 功能 |
|------|------|
| ↑ / W | 加速 |
| ↓ / S | 刹车 / 倒车 |
| ←→ / AD | 转向 |
| Shift | 漂移 (转向时按住) |
| R | 重玩 |
| Q | 退出 |
| SPACE | 开始游戏 |

## UI 设计

### HUD 布局

```
┌─────────────────────┐
│                     │
│         [████░░░░]  │  ← 漂移蓄力条
│                     │
│         120 km/h    │  ← 当前速度
│                     │
│  TIME: 00:00.000    │  ← 当前用时
└─────────────────────┘
```

### 视觉风格
- 赛道：深灰色背景 + 浅灰可行驶区域 + 白色边界线
- 赛车：简单矩形或色块，颜色区分（蓝色正常 / 漂移时紫色 / 喷射时红色）
- 漂移条：渐变色（蓝→绿→黄→红）
- 字体：等宽字体，清晰可读
